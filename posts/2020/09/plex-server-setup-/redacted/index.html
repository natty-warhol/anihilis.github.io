Plex Server Setup //Redacted//

A Walkthrough

I have finally found my perfect Plex setup, as shown in my post about setting up Artix (the subcategory about Servers is the example). My preferred setup is a boxed in Plex setup, to prevent access unwanted access from the UPnP ports.

Beginning Notes

I believe we can collect installation notes from the https://natty-warhol.github.io/posts/2020/08/artixandrunit-install/. I prefer to avoid encryption so I can restart the computer remotely.

Mapping looks something like this:

/dev/sda

/dev/sda1 -> /boot
/dev/sda2 -> /boot/efi
/dev/sda3 -> /
/dev/sda4 -> SWAP
/dev/sdb1 -> /home/plex/media/Movies
/dev/sdc1 -> /home/plex/media/TV
/dev/sdd1 -> /home/plex/meida/Music

User Accounts Set Up:

You should only ever use root for administration and installation purposes. So in this case, we are setting up a very hard password for it, and writing it down somewhere to never be used again.

I like to give myself a sudo-able user for SSHing into with, when setting up accounts, this is the only one that should have sudo permissions. Following the creation of the administrative account, we can go on to the Plex account.

With the media/Plex account, I like to give it almost no permissions, I give it access to read the media hard drives (which should be mounted to this account anyways), and this is where the docker server will be planted.

Good Measures

Before setting up any port forwarding or any internet access, I like to set up security measures. There will be no VPN as this will cause my ports to be blocked from plex. Instead, I change the DNS either to 1.1.1.1 or Quad9, as well as set up UFW.

With UFW I set the defaults to deny all incoming except for a designated plex and SSH port (which should be changed from the default ports to at least 4 digit ports (5 in Plex’s case)

After this, Things should be fine, all the internet access that will be allowd to this device anyway will be limited.

Plex Installation

As the administrative user, go ahead an install docker as well as the docker init script (runit for me).

pacman -S docker docker-runit  
ln -s /etc/runit/sv/docker /run/runit/service

Docker install and set-up instruction can be found here.

Required commands to remember after setup are:

docker start plex  
docker stop plex

Simply for restarting and updating.

With the commands run from that website, the VM will only be able to read the files on the media drives, and no other drives on the system.

Voila!

P.S.

After you’re all done, there are some tools I can recommend for you!
Since the server will basically only be able to read the media files, I have ffmpeg set up with H.265 for compression when needed through the administrative account.

And for health checking I use glances mostly (it shows the temps).

For media transfers, I recommend Syncthing, as it will allow you to use local internet. Just remember to check your sync options, I always feel more comfortable when my sync folders are one way read/sync.
