In my Many Journeys

I find myself hopelessly trying to find the cleanest most efficient way to get the things I want done, done. I have hopes to maybe make another one of these soon, instead using FreeBSD as hopefully a final home for all my server needs, but until I become comfortable with that world, I’m going to focus on using Linux in the most organized way possible.

A quick explanation:

If you’re a regular user of linux and partake in the social media and the peoples involved with the Linux community, you probably have heard about people wanted the smallest installs with the least amount of packages and so forth and so forth. I have a similar neuroticism, but mine is simply that of organization.
I feel that linux has a habit of cluttering itself over time. I prefer to know what is on my system, and I don’t want random cruft building up over time, be it old packages (like in Arch how it holds onto previous versions of packages), or a loss of understanding the resources set up (human knowledge error). Linux honestly has too many options, not like GNU options (one package/piece of code for action), but just on the system already. I want a system where I can do everything with single commands. In Linux these days, there’s kernel supported legacy items, as well as the all encompassing, always install SystemD. For instance, cron jobs were a thing that people used to do (and a lot still do), but SystemD, which is installed on all major distributions, also have a timer system.
I have remedied this mostly with my current laptop setup (because I would murder someone if I had to live knowing all that extra garbage was on my system) by simply using only SystemD for all my needs, from mounting, to timers, to user management. I hate it, but it’s better that just using it to boot and other stuff for other things, while I have all this unused code on my kernel and in my system.
With the BSDs of the world, they are very small, the kernel only including things you absolutely need, and the software count is limited to ports from linux and anything the devs feel is absolutely important. It’s also nice how everything is cohesive and in order, from the locations of the binaries to the settings, and GODDAMN do I love how there’s never more than two solutions to one problem. I STG I’m tired of having so many ways to fix one problem on my system, and knowing that I can go around and accidently do one of those other things not knowing I’m screwing with another part of my system!
Anyways, I’m done ranting now, I got a little keyboard rage there….

User Accounts

This time around I was lacking in inspiration and took to the internet. Decided to name things after Lovecraftian places and creatures.. Which kinda fits my solar theme with my mobile devices.
Also this time around, I decided to scrap the multiple users with the Docker BS (I grew tired of Docker, with Plex, you give up a lot of accessibility features). Instead I went ahead with creating one user with root priv. and another user without. I’m SSHing into the non-rootable user, and then using the su command.
I’m doing this to deal with the confusion, I also ended up having trials and errors with ownership privileges, and transferring files between accounts was a nightmare.
The rootable account has all the services in it’s name, and I can only perform maintenance remotely (unless remoting in with teamviewer).

Internet Access and Security

This time around, I went the easy way with Debian and XFCE as a desktop right out the box. I’m using the standard gui NetworkManager connected to Ethernet (with a simultaneous WiFi for the VM we’ll talk about later). I have it set up to access the Quad9 DNS only (no more 1.1.1.1).
For the firewall I’m still using the trusty UFW (fuck iptables, and thank god for simplicity of use). I stick with the default of deny, and a limit on 64 (my SSH port of choice).

Plex Installation

Luckily, Debian has a PPA for direct Plex updates! just go to the website, install the deb file they provide, and then the mirror comes with it. Wamo-bamo.

Virtual Machine for Media Collection

In order to offload all the work I do for the server on my laptop I decided to turn my server into two computers instead of just getting a cheap other one. I’m using Virtualbox (My server is a Galago Pro, it can’t even VM right, I’m not gonna waste my time with QEMU for that. I’ve set the network up to bridged, and have my private VPN service that I connect to from the VM, and then can SSH into it from my laptop or the other server to use any of the services I choose to for collection.
Generally speaking, I will prefer to use Arch or something derivative for this, just because updates are good for security, especially if you’re gonna be doing a lot of downloading other’s materials.
